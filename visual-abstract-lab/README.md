# Visual Abstract Lab

## Für Lehrende auf einen Blick

**Dieses Spiel ist geeignet wenn Sie in Ihrer Lehrveranstaltung
folgende Themen behandeln**

Visual Abstracts als grafische Kurzfassung eines Forschungsartikels.
Was ist ein Visual Abstract und wofür wird er gebraucht. Die sechs
Design-Prinzipien eines guten Visual Abstracts. Der Gestaltungs-
Workflow vom Artikel zum Visual Abstract.

**Typische Einsatzszenarien**

Schreibwerkstätten, Seminare zur Wissenschaftskommunikation,
Begleitlehrveranstaltungen zur Masterarbeit oder Dissertation,
Methodenkurse, Module zu Open Science und Forschungsmarketing,
Workshops zur Sichtbarkeit eigener Forschung.

**Sprache:** Vollständig auf Deutsch. Keine Englischkenntnisse
notwendig. Inhaltliche Grundlage ist die Elsevier Reihe "From
Article to Art".

**Dauer:** Etwa 20 bis 30 Minuten Spielzeit. Mastery Score liegt bei
45 Prozent.

**Voraussetzungen für Studierende:** Keine. Vor jeder Spielrunde
liefert ein eigener Theorie-Screen den nötigen Input, das Spiel
funktioniert daher auch ganz ohne Vorwissen.

## Was lernen die Studierenden

Nach dem Spielen können die Studierenden erklären was ein Visual
Abstract ist und welche kommunikative Funktion er erfüllt. Sie
kennen die sechs Design-Prinzipien eines guten Visual Abstracts und
erkennen typische Qualitätsprobleme in fremden Visual Abstracts. Sie
verstehen den Gestaltungs-Workflow vom wissenschaftlichen Artikel
zum fertigen Visual Abstract und können erste eigene Skizzen
strukturiert anlegen.

## Wie wird gelernt

Visual Abstract Lab ist ein dreistufiges Lernspiel mit dem Avatar
Prof. Dr. Editor als narrativer Begleitfigur. Jede Runde wird durch
einen eigenen Theorie-Screen eingeleitet, der die Lernziele klar
benennt und das nötige Vorwissen aufbaut. Erst danach folgt die
spielerische Anwendung.

Runde 1 (Verstehen): Drei Visual Abstracts werden vorgestellt, die
Studierenden ordnen sie ein und beantworten Fragen zur Funktion und
Wirkung.

Runde 2 (Anwenden): Ein professionell wirkender Visual Abstract
enthält versteckte Qualitätsprobleme. Die Studierenden klicken auf
die Bereiche, die ihnen verdächtig vorkommen, und erhalten erklärendes
Feedback.

Runde 3 (Visual Abstract Builder): In einem geführten Schritt-für-
Schritt-Prozess setzen die Studierenden einen eigenen Visual Abstract
zusammen.

Das Feedback ist durchgehend erklärend statt nur bewertend, jede
Runde hat klar benannte Lernziele. Diese didaktische Haltung war der
Gruppe besonders wichtig, damit das Spiel auch von Studierenden ohne
jegliches Vorwissen erfolgreich durchgespielt werden kann.

## SCORM Paket herunterladen

Die Datei `visual-abstract-lab.zip` in diesem Ordner ist ein
vollständiges SCORM 1.2 Paket mit imsmanifest.xml, API-Wrapper und
Tracking-Logik und kann direkt in Moodle als Aktivität "Lernpaket
(SCORM)" hochgeladen werden. Punktestand und Lernfortschritt werden
automatisch erfasst. Mastery Score liegt bei 45 Prozent. Das Spiel
ist responsive und kommt ohne externe Bibliotheken aus.

## Entstehungskontext

Entwickelt im Rahmen der Lehrveranstaltung von Barbara Geyer im
Masterstudiengang E-Learning und Wissensmanagement (MEWM) an der
Hochschule Burgenland, Sommersemester 2026.

## Verwendete KI Tools und Vorgehen

Der Entwicklungsprozess war modellgetrieben und wechselhaft. Zu
Beginn wurde ChatGPT verwendet, um aus Konzept und
Inhaltszusammenfassung der Elsevier Module HTML, CSS und JavaScript
zu generieren. Der Output war strukturiert, die Integration in eine
lokale Ordnerstruktur jedoch aufwändig. Im zweiten Schritt wurde
ChatGPT Codex über einen geteilten OneDrive Ordner eingesetzt. Das
führte zu einem Kontrollverlust, weil Codex eigenständig
Fragestellungen, Feedbacktexte und Spiellogik überschrieb.

Der Wechsel zu Claude markierte die Wende. Im regulären Chat-Modus
lieferte Claude beim ersten Versuch ein spielbares, visuell
ansprechendes Ergebnis mit allen drei Rundentypen. Iterative
Anpassungen wurden präzise und ohne unerwünschte Seiteneffekte
umgesetzt. Zum Testen wurde SCORM Cloud als Hosting Plattform
genutzt, was eine realitätsnahe Überprüfung vor dem Moodle Upload
ermöglichte.

## Reflexion zur KI Unterstützung

Claude erwies sich als besonders stark bei der Umsetzung komplexer,
mehrstufiger Anforderungen. Das vollständige SCORM Paket mit
imsmanifest.xml, API Wrapper und Tracking Logik, die Web Audio API
Integration, das responsive Design mit Breakpoints sowie der SVG
Avatar wurden ohne externe Bibliotheken erstellt.

Die Grenzen lagen im Bereich visuelles Feintuning. Schriftgrößen,
Abstände und Farbkontraste erforderten mehrere Anpassungsschleifen,
weil das tatsächliche Rendering am Bildschirm von der KI nicht
direkt überprüft werden kann. Hier bleibt menschliches Urteilsver-
mögen unverzichtbar.

Die wichtigste didaktische Erkenntnis: Ein Lernspiel braucht vor
jeder Spielrunde einen gezielten Theorie Input. Ohne diesen Vorlauf
fühlten sich die Aufgaben unvorbereitet an, selbst für Teilnehmende
mit Vorkenntnissen. Ein gut gestaltetes Lernspiel muss so konzipiert
sein, dass auch jemand ohne jegliches Vorwissen erfolgreich
durchspielen kann. Der Avatar Prof. Dr. Editor verleiht dem Spiel
eine narrative Identität und macht aus einer Abfolge von Aufgaben
ein zusammenhängendes Erlebnis.
